图片插入测试

发表于 2019-05-26

文章里很多需要插入图片，一般都是写文章时现场截的图，文章里图片插入要提供图片url，但是本地的图片要上传到网上才有地址，很不方便。

搜了一下，hexo有上传本地图片的方法。

  1. 把博客配置文件_config.yml修改一个地方：post_asset_folder:true
  2. 安装插件：npm install hexo-asset-image –save

然后新建文章hexo new “img-test”，会在文章md文件目录下新建”img-test.md”和”img-test”文件夹，可以将图片放在文件夹里，引用时地址填：img-test/图片名.jpg

文章上传到git上后图片就有了，这个方法引用时也非常方便，很好用呢！

参考链接：https://www.jianshu.com/p/a2786cdb06e1

几个常见的在x=0处展开的带佩亚诺余项的泰勒公式：

$e^x = 1+x+\frac{1}{2!}x^2+…+\frac{1}{n!}x^n+o(x^n)$

$sin(x) = x-\frac{1}{3!}x^3+…+\frac{(-1)^n}{(2n+1)!}x^{2n+1}+o(x^{2n+2})$

$$cos(x) = 1-\frac{1}{2!}x^2 + … + \frac{(-1)^n}{(2n)!}x^{(2n)} + o(x^{2n+1})$$

$$ln(1+x) = x - \frac{x^2}{2} + \frac{x^3}{3} - … + (-1)^{n-1}\frac{x^n}{n} + o(x^n)$$

$$(1+x)^m = 1+mx + \frac{m(m-1)}{2!}x^3 + … + \frac{m(m-1)…(m-n+1)}{n!}x^n + o(x^n)$$

测试markdown语法

发表于 2019-05-25

测试markdown语法

通过什么来测试md语法呢，就随便写点内容吧。

二级标题，我是二级标题

​ 自我理解，md是是为了开发者能更简单的设计出网页而设计的，直接写html比较心累，我不会写，哈哈，我只能看懂网页的源码，或者简单修改，让我徒手撸网页还真撸不出来。 由于md非常方便，写起来简单，并且转换成网页或者PDF后很直观，一些开发者就增加了它的格式，像cmd-markdown就支持流程图、latex数学公式这些高级东西，但是我在其他md测试的地方就不支持，有的渲染不出来。
这些高级东西，感觉md需要有一个统一的标准。

ps：

md两个格式之间要有一个换行，md里段落换行有点怪

不同工具转换的格式不同可真不舒服，我用notepad++插件 ‘>’ 竟然不能转换，有的格式会渲染不出来

还有空格，多个空格好像只能显示一个，那段首2个空格怎么显示呢

一些问题写的多了应该就知道了，这篇只是个测试md语法的，以后再写个详细的。

我还是二级标题

  • 小生不才，github上没有发过东西，写过一些代码也都是学习的时候写的，有些自己感觉写的还行，但都是写完用过就放那了，啥时候要用了再找回来用。很多写的也没规范，注释都不全，以至于再翻出来用的时候还要想一遍代码的逻辑，逻辑有点复杂的就要像很久。过错过错，做一套六级题谢罪。

  • 以前在服务器上搭过博客，因为太懒没维护，就荒了，后来用于微信公众号后台，然后就没后来了。在博客园上发过一些学习性的文章，偶尔看到github.io上可以放静态博客，看着还挺好，就整了一个，here，它是通过hexo将md文件渲染成html，上传到git上访问的，才有了这篇无聊的文章。

  • hexo好像有不同的渲染器，它们支持的md语法有所不同，有的可以渲染数学公式、流程图感觉很不错，有时间试试。

md编辑（三级标题，我是三级标题）

网上有很多md在线编辑器：

  1. http://tool.oschina.net/markdown/
  2. http://md.geekbang.org/v2/
  3. https://www.zybuluo.com/mdeditor

第三个是cmd-markdown，可以写出很好看的文章，然后可以导出html或者pdf，估计它的md文件有的效果hexo渲染不出来。

另外还有很多md编辑软件，我用的是typora，感觉很好用。

二级标题

我在网上看到一篇文章里说”看的越多，不会的越多”，挺有道理的，了解了很多，看似什么都知道，但要是让给别人讲，却什么也讲不出来。感觉我就是这样。

知识是需要沉淀的，现在很多微信公众号推很多技术文章，浏览一遍就感觉自己懂得了，其实并没有。写博客挺有意义的，能将自己懂得的输出给别人自己会有更深的理解。记笔记是不能与文章比的，我以前以为把知识随笔记下，以后可以看就能够掌握，可是笔记写的太随意，并且不够全面，没写一篇文章理解的深。
